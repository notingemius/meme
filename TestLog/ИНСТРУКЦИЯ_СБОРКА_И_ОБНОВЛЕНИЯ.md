# 📦 Подробная инструкция: собрать APK + обновления «по воздуху» (с нуля)

Эта инструкция для человека, у которого **нет никаких аккаунтов** (кроме GitHub)
и который раньше этого не делал. Делай по шагам сверху вниз.

---

## Что понадобится

- **Компьютер** (Windows, Mac или Linux). Команды вводятся в терминале.
- **Телефон Android** (твой OnePlus 7).
- **Интернет**.
- **Аккаунт GitHub** — у тебя уже есть (`notingemius`).
- **Бесплатный аккаунт Expo** — создадим в Шаге 2 (это бесплатно).

> ⚠️ Сама сборка APK идёт в облаке Expo, но команды нужно запускать
> с компьютера. С одного телефона это не сделать.

---

## Шаг 1. Установить Node.js (если ещё нет)

1. Открой https://nodejs.org
2. Скачай версию **LTS** (большая зелёная кнопка) и установи (просто «Далее → Далее»).
3. Открой терминал:
   - **Windows**: меню Пуск → набери `PowerShell` → Enter.
   - **Mac**: Spotlight (⌘+Пробел) → `Terminal` → Enter.
   - **Linux**: твой терминал.
4. Проверь, что Node установился — введи по очереди:
   ```bash
   node -v
   npm -v
   ```
   Должны показаться номера версий (например `v20.x.x` и `10.x.x`).

---

## Шаг 2. Создать бесплатный аккаунт Expo

1. Открой https://expo.dev
2. Нажми **Sign Up** (Регистрация).
3. Введи **email**, придумай **username** (логин) и **пароль**. Запиши их.
4. Зайди в почту, открой письмо от Expo и подтверди email (кнопка/ссылка).

Готово. Аккаунт бесплатный, его хватает с запасом (≈30 сборок в месяц + обновления).

---

## Шаг 3. Установить инструмент EAS

В терминале введи:
```bash
npm i -g eas-cli
```
Проверь:
```bash
eas --version
```
(если показал версию — всё ок).

> Если на Mac/Linux ошибка прав — добавь `sudo` в начало:
> `sudo npm i -g eas-cli`

---

## Шаг 4. Скачать код игры

**Вариант А — через git** (если установлен git):
```bash
git clone https://github.com/notingemius/meme.git
cd meme
git checkout real-memes-multiplayer
cd TestLog
```

**Вариант Б — без git (скачать ZIP):**
1. Открой https://github.com/notingemius/meme/tree/real-memes-multiplayer
2. Зелёная кнопка **Code** → **Download ZIP**.
3. Распакуй архив.
4. В терминале зайди в папку `TestLog` внутри распакованного проекта, например:
   ```bash
   cd путь/к/meme/TestLog
   ```

> Если ты уже смержил Pull Request в `main`, можешь брать ветку `main`.

---

## Шаг 5. Установить зависимости проекта

Находясь в папке `TestLog`:
```bash
npm install
```
(подожди минуту, скачаются библиотеки).

---

## Шаг 6. Войти в свой аккаунт Expo

```bash
eas login
```
Введи **email/username и пароль** из Шага 2.

---

## Шаг 7. Добавить нативные модули (Wi-Fi + обновления)

```bash
npx expo install expo-updates react-native-tcp-socket expo-network
```
> `expo install` сам подберёт версии, совместимые с твоим проектом.

---

## Шаг 8. Привязать проект к твоему аккаунту

```bash
eas init
```
- Согласись создать проект (нажми Enter / `y`).
- Это создаст уникальный `projectId` и запишет его в `app.json`.

---

## Шаг 9. Настроить обновления «по воздуху»

```bash
eas update:configure
```
- Это автоматически заполнит `app.json` (заменит `REPLACE_WITH_PROJECT_ID`
  на твои реальные значения и пропишет адрес сервера обновлений).

---

## Шаг 10. Собрать APK (один раз, ~10–15 минут)

```bash
eas build -p android --profile preview
```
- Подтверди вопросы (Enter).
- Сборка идёт в облаке Expo. Когда закончится — в терминале появится
  **ссылка на готовый APK** (и она же будет в твоём кабинете на expo.dev → Builds).

---

## Шаг 11. Установить APK на телефон

1. Открой ссылку из Шага 10 **прямо в браузере телефона** (или скинь файл на телефон).
2. Скачай `.apk`.
3. При установке телефон попросит разрешить **«установку из неизвестных источников»** — разреши.
4. Установи и запусти приложение **МемКарти**.

✅ Теперь работает: реальные мемы, игра с ботами, игра с другом по Wi-Fi, кнопка «Обновить».

---

## Шаг 12. Как обновлять приложение ПОТОМ (по воздуху, без пересборки)

Каждый раз, когда поменял код игры (логика, тексты, картинки):
```bash
cd TestLog
eas update --branch preview -m "что я поправил"
```
Затем на телефоне в меню жми **«⟳ Оновити застосунок»** — прилетит новая версия.
APK заново собирать **НЕ нужно**.

---

## Когда всё-таки нужна пересборка APK (Шаг 10 снова)

Только если:
- добавил **новый нативный модуль** (новую библиотеку с native-кодом),
- поменял **версию Expo / React Native**,
- сменил **иконку, имя пакета, разрешения**.

Тогда: подними `version` в `app.json` (например `1.0.0` → `1.0.1`) и снова `eas build`.

---

## Частые вопросы

**«Это бесплатно?»** — Да. Бесплатного тарифа Expo хватает.

**«Нужен ли компьютер?»** — Да, для команд `eas`. Сборка идёт в облаке.

**«Wi-Fi-игра работает без сборки (в Expo Go)?»** — Нет. Только в собранном APK
из Шага 10, потому что это нативный код.

**«Я запутался на каком-то шаге»** — напиши мне номер шага и текст ошибки
из терминала, я подскажу.
