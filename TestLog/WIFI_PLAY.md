# Гра з другом по Wi-Fi (локальна мережа)

Два телефони грають разом **без сервера та без інтернету** — напряму по Wi-Fi
через TCP-сокети. Один телефон стає **хостом** (тримає логіку гри), другий
**приєднується** за IP хоста.

## Як це працює

- Меню → **«📶 Гра з другом (Wi-Fi)»** → екран вибору.
- **Створити гру** (хост): відкриває TCP-сервер на порту `8787`, у лобі
  показує свій IP. Хост теж грає (локальний клієнт через `127.0.0.1`).
- **Приєднатися**: вводиш IP хоста → підключаєшся.
- Обидва телефони мають бути в **одній Wi-Fi-мережі** (або роздай точку доступу
  з телефона-хоста і підключи до неї другий телефон).
- Для старту в режимі «суддя» потрібно **3+ гравці** — якщо вас двоє, хост
  тисне **«+ Додати бота»** у лобі.

## ⚠️ Потрібна пересборка APK (це native-модуль)

Wi-Fi працює на TCP-сокетах (`react-native-tcp-socket`) — це нативний код,
тому **в Expo Go не працює** і **по OTA не прилітає**. Потрібно один раз
зібрати APK з цим модулем:

```bash
cd TestLog
npx expo install react-native-tcp-socket expo-network   # правильні версії під SDK
npx expo prebuild -p android --clean
# або зібрати в хмарі:
eas build -p android --profile preview
```

Локальна збірка (якщо є Android SDK + JDK 17):
```bash
cd android && ./gradlew assembleRelease -PreactNativeArchitectures=arm64-v8a
```

> Після цього **звичайні зміни коду** все одно летять по повітрю (`eas update`).
> Пересборка потрібна лише через те, що ми **додали новий нативний модуль**.

## Код

- Транспорт: `TestLog/src/game/net/lanLink.ts` (TCP-сервер/клієнт + `getLocalIp`).
- Хук матчу: `TestLog/src/game/useMatch.ts` (боти / wifi-host / wifi-join).
- Екран вибору: `TestLog/src/screens/WifiSetupScreen.tsx`.
- Банер з IP / статусом: `TestLog/src/screens/PlayScreen.tsx`.

Архітектура transport-agnostic: та сама логіка гри (`OfflineHost`/`OfflineClient`)
працює і з ботами (in-memory), і по Wi-Fi (TCP) — змінюється лише «труба».
