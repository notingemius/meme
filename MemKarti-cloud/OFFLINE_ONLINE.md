# МемКарти — гра по Wi-Fi (офлайн) і онлайн

Цей документ пояснює, як влаштована гра **поряд по Wi-Fi** (без сервера) і **онлайн** (через бекенд), і як зібрати робочий білд.

---

## 1. Два режими гри

| Режим | Як працює | Інтернет |
|------|-----------|----------|
| **Поряд по Wi-Fi** | Один телефон — *хост*. Він тримає всю логіку гри і відкриває TCP-сокет. Інші телефони підключаються по локальній мережі (сканують QR з IP або вводять IP вручну). | Не потрібен. Лише спільний Wi-Fi. |
| **Онлайн** | Усі підключаються до бекенда (Next.js + Neon). Стан зберігається на сервері, клієнти опитують `/state`. | Потрібен. |

Обидва режими використовують **один ігровий рушій** (`apps/mobile/src/game/engine`) і **один інтерфейс клієнта** `GameClient` (`apps/mobile/src/game/net`). Екрани не знають, локальна це гра чи онлайн — різниця лише у фабриці `createClientForMode`.

---

## 2. Архітектура мережевого шару

```
src/game/
  engine/        # чистий рушій гри (без мережі) — детермінований, покритий симуляціями
    engine.ts
    deck.ts
    engine.sim.ts
  net/
    protocol.ts  # формат повідомлень hello/welcome/state/action/error (+ encode/decode)
    link.ts      # абстракція транспорту (ServerLink/ClientLink) + InMemory для тестів
    lanLink.ts   # РЕАЛЬНИЙ Wi-Fi-транспорт (TCP-сокети) + getLocalIp()
    offline.ts   # OfflineHost (авторитетний стан) + OfflineClient (реалізує GameClient)
    online.ts    # OnlineClient — REST-клієнт до бекенда
    index.ts     # createClientForMode(): online | wifi-host | wifi-join
    net.sim.ts     # інтеграційний тест мережі (повний матч, анонімність)
    timeout.sim.ts # тест тайм-ауту ходу (хост доходить за AFK-гравців)
```

**Ключові властивості:**
- **Анонімність** збережена: хост розсилає кожному гравцю його *персональний* вид (`viewFor`), руки інших і автори мемів приховані на клієнті.
- **Перепідключення**: якщо телефон гравця втратив Wi-Fi, `OfflineClient` сам перепідключається (експоненційний backoff) і відновлює сесію за `resumeToken` — гравець повертається у ту саму гру.
- **Тайм-аут ходу (60с)**: якщо гравець завис або вийшов, хост автоматично доходить за нього (випадковий мем / суддя / голос), щоб гра не зависала.

---

## 3. ⚠️ Важливо: потрібен власний білд (dev-build або APK)

Режим «поряд по Wi-Fi» використовує **нативні модулі** (TCP-сокети, камера, мережа). Тому гру треба зібрати як **development build** або **APK** — звичайний JS-прев'ю без нативних модулів не підійде (див. розділ 5).

---

## 4. Встановлення нативних залежностей

`expo-camera` вже є у проєкті. Додатково встанови:

```bash
cd apps/mobile

# мережеві модулі під Expo SDK 54
npx expo install expo-network

# TCP-сокети та QR (не керуються expo install — став через yarn)
yarn add react-native-tcp-socket react-native-qrcode-svg
```

| Пакет | Для чого |
|-------|----------|
| `react-native-tcp-socket` | TCP-зʼєднання між телефонами по Wi-Fi |
| `react-native-qrcode-svg` | хост показує QR з IP (потребує `react-native-svg`, вже є) |
| `expo-camera` | сканування QR на телефоні гравця (вже у залежностях) |
| `expo-network` | визначення локального IP хоста |

> Код написаний так, що якщо якогось пакета немає — застосунок **не падає**: QR замінюється текстовим IP, а сокети дають зрозуміле повідомлення про помилку. Але для реально�� гри по Wi-Fi всі чотири пакети потрібні.

Дозволи вже додані в `app.json`: `CAMERA`, `INTERNET`, `ACCESS_WIFI_STATE`, `ACCESS_NETWORK_STATE` + плагін `expo-camera`.

## 5. Збірка

```bash
cd apps/mobile

# development build (рекомендовано для тестів Wi-Fi)
eas build -p android --profile development

# або готовий APK для встановлення на телефони
```

Встанови отриманий `.apk` (або dev-build) на 2–3 телефони.

## 6. Як грати по Wi-Fi

1. Усі телефони підключаються до **однієї Wi-Fi-мережі**.
2. На головному екрані тисни **«Рядом по Wi-Fi»**.
3. Один гравець обирає **«Створити гру (я — хост)»** — зʼявляється **QR-код** і IP-адреса.
4. Інші обирають **«Приєднатися»** → сканують QR (або вводять IP вручну).
5. Коли зібралось **≥ 3 гравці** (можна додати ботів), хост тисне **«Почати гру»**.

### Якщо телефони не бачать одне одного
У багатьох публічних/гостьових Wi-Fi увімкнена **ізоляція клієнтів** — пристрої не можуть зʼєднатися напряму. Рішення:
- роздай **точку доступу (hotspot)** з телефона-хоста, і нехай інші підключаться до неї;
- або використовуй домашній роутер без ізоляції.

## 7. Тести (запускаються в Node, без телефона)

Логіка рушія та мережі покрита детермінованими симуляціями:

```bash
# рушій гри (повні матчі, режими суддя/голосування)
npx tsx apps/mobile/src/game/engine/engine.sim.ts

# мережевий шар (hello/welcome, розсилка, анонімність, повний матч)
npx tsx apps/mobile/src/game/net/net.sim.ts

# тайм-аут ходу (хост доходить за AFK-гравців)
npx tsx apps/mobile/src/game/net/timeout.sim.ts
```

> Нативні модулі (сокети/камера) не тестуються в Node — їх перевіряємо вже на dev-білді. Симуляції використовують `InMemoryLink`, який емулює транспорт у памʼяті.

## 8. Онлайн-режим (нагадування)

Для онлайну потрібен запущений бекенд (`apps/web`) і змінні оточення мобільного застосунку:
`EXPO_PUBLIC_BASE_URL` / `EXPO_PUBLIC_PROXY_BASE_URL`. Деталі — у `SETUP.md`.
